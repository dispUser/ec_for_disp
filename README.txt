■全体像
ソースは大きく３つに分かれます。
①通販システム(src/front)
②管理システム(src/admin)
③自動単体テスト(test)

■通販システムで出来ること
・注文レコードの作成
・会員登録
・注文履歴の閲覧
・注文ステータスの更新（入金済)

注文フローは以下になります。
・商品検索
・カート投入
・会員登録
・購入（注文レコードの作成）
・入金更新
・キャンセル

※振込は外部サイトを使いますので、
決済システムはありません。

■管理システムで出来ること
受注管理とマスタ管理を行います。

受注管理では、以下が可能です。
・注文履歴の閲覧
・受注ステータスの更新(発送済、キャンセル)

マスタ管理では、以下のマスタの編集が可能です。
・商品マスタ
・会員マスタ
・注文マスタ
・管理者マスタ

■自動単体テストで出来ること
注文フローの単体テストコードを作成しています。
一部、管理システムの単体テストコードもあります。
作成者が私１人のため、時間を有効活用するため、
テスト自動化を導入しました。

■ECサイトの作成状況
注文から受注処理まで一通り、動きます。

商品登録時、1商品に複数の画像を登録したいという要望が出ました。
商品データの持ち方が変わるので、全面的に見直し中です。

CSSの設計が出来ておらず、管理の難しいCSSになりました。
BEMを学習し、一部置き換え済です。

今後も学習、リファクタを続けていく予定です。

■運用状況
商品登録機能の見直しをしているため、まだ稼働していません。
商品登録の見直しが終われば、Facebookで告知し、運用開始します。
